# Test Vector Provenance

All vectors were copied into this tree on 2026-03-21 so that tests
are reproducible without network access. This file records where each
set came from and how to refresh it.

## shavs/

NIST SHAVS (SHA Validation System) response files.

- **Origin:** NIST CAVP "Secure Hash Algorithm Validation System (SHAVS)"
  https://csrc.nist.gov/projects/cryptographic-algorithm-validation-program/secure-hashing
- **Copied from:** `wolfcrypt-ring-compat/third_party/NIST/SHAVS/` (which mirrored
  the NIST download unchanged).
- **Files:** `SHA{1,224,256,384,512}{ShortMsg,LongMsg,Monte}.rsp`,
  `SHA3_{256,384,512}{ShortMsg,LongMsg,Monte}.rsp`
- **Generated by NIST:** 2011-03-15 (SHA-2), various dates (SHA-3).

To update: download fresh `.rsp` files from the NIST CAVP page above
and drop them into this directory, replacing the existing files.

## cavp/

NIST CAVP (Cryptographic Algorithm Validation Program) CMAC test vectors.

- **Origin:** NIST CAVP "Block Cipher Modes — CMAC"
  https://csrc.nist.gov/projects/cryptographic-algorithm-validation-program/block-ciphers
- **Standard:** NIST SP 800-38B (Recommendation for Block Cipher Modes
  of Operation: the CMAC Mode for Authentication).
- **Copied from:** `wolfcrypt-ring-compat/tests/data/`
- **Files:**
  - `cavp_aes128_cmac_tests.txt` — AES-128 CMAC verify vectors
  - `cavp_aes192_cmac_tests.txt` — AES-192 CMAC verify vectors
  - `cavp_aes256_cmac_tests.txt` — AES-256 CMAC verify vectors
  - `cavp_3des_cmac_tests.txt` — 3DES CMAC verify vectors
- **Generated by NIST:** 2011-03-15.

To update: download the CMACVer response files from the NIST CAVP page
and replace the corresponding `.txt` files here.

## acvp/

NIST ACVP (Automated Cryptographic Validation Protocol) test vectors
for ML-DSA (FIPS 204, Module-Lattice-Based Digital Signature Standard).

- **Origin:** NIST ACVP server (demo instance).
  https://pages.nist.gov/ACVP/
- **Standard:** NIST FIPS 204.
- **Copied from:** `wolfcrypt-ring-compat/tests/data/`
- **Files:**
  - `MLDSA_{44,65,87}_ACVP_keyGen.txt` — deterministic key generation vectors
  - `MLDSA_{44,65,87}_sigVer.txt` — signature verification vectors
- **Note:** these vectors are not yet wired into the test suite (see
  `tests/mldsa.rs` header comment for the reasons).

To update: request fresh vectors from the NIST ACVP demo server for
the ML-DSA algorithm family and convert to the text format used here.
